## Error Logs & Resolutions

### Issue 1: `sd-server` Startup with Invalid Arguments
- **When**: First manual restart of `sd-server`.
- **Symptom**: `image-gen.log` printed the help text (`--help`) instead of loading the model.
- **Root Cause**: Supplied FLUX.1 Schnell arguments (`--diffusion-model`, `--clip_l`, `--t5xxl`, `--vae`) instead of `-m` required for SD1.5.
- **Resolution**: Aligned command with `verify-host-services.sh` (line 205): 
  `"$SD_SERVER_BIN" --listen-port 8086 -m "$SD_MODEL" --seed -1`

### Issue 2: Health Check Timeout (30s insufficient)
- **When**: Third E2E test execution.
- **Symptom**: `Banned Image Gen Worker did not become healthy within 30s.`
- **Root Cause**: The `sd-server` was restarted 15s prior to the E2E run. Loading 1,131 tensors into memory took 4.68s since system memory pages were previously purged. 
- **Resolution**: Triggered subsequent E2E runs only after validating `sd-server` responsiveness via `curl -s http://localhost:8086/`.
